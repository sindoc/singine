# iOS / Android Connection Guide — singine local network server

## Overview

When `singine serve` is running on your Mac at `192.168.200.104:8080`, all iOS and
Android devices on the **same local network** can connect to it directly.

No VPN, no cloud, no auth token needed for read-only endpoints (`/health`, `/cap`).

---

## Mac: start the server

```bash
cd ~/ws/singine

# Option A: Python CLI (detects capabilities, then starts Clojure server)
singine serve

# Option B: direct Clojure (faster, no detection)
cd core && clojure -M:serve

# Option C: dry-run (no real mail/Kafka — safe for testing)
singine serve --dry-run
```

The server prints your local IPs on startup:
```
  Connect from your iOS devices (same LAN):
    iPad (iSH):        curl http://192.168.200.104:8080/health
    iPhone (a-Shell):  curl http://192.168.200.104:8080/health
    Android (Termux):  curl http://192.168.200.104:8080/health
```

---

## iPad (iSH — Alpine Linux)

iSH provides a Linux shell inside iOS. Supports `curl`, `python3`, and `apk`.

```sh
# Install curl if not present
apk add curl

# Ping the server
curl http://192.168.200.104:8080/health

# Get machine capability profile
curl http://192.168.200.104:8080/cap | python3 -m json.tool

# Query timezone
curl "http://192.168.200.104:8080/timez?cities=BRU,NYC"

# Resolve a location
curl http://192.168.200.104:8080/loc/BRU

# Search mail
curl "http://192.168.200.104:8080/messages?search=invoice"
```

**Note on iSH networking**: iSH uses localhost tunnelling. If `192.168.200.104` doesn't
work, try the Mac's link-local IPv6 address or use a Bonjour name (see below).

---

## iPhone (a-Shell)

a-Shell provides a Unix-like shell on iOS with built-in `curl` and Python 3.

```sh
# Ping
curl http://192.168.200.104:8080/health

# Capability profile (pretty-printed)
curl -s http://192.168.200.104:8080/cap | python3 -c "import sys,json; print(json.dumps(json.load(sys.stdin), indent=2))"

# Location-action correlation
curl -s http://192.168.200.104:8080/loc/BRU

# Timezone query
curl -s "http://192.168.200.104:8080/timez?cities=BRU,LON,KBL"
```

## Android (Samsung S8 — Termux)

Termux provides a full Linux environment on Android.

```sh
# Install curl if needed
pkg install curl

# Connect to singine server
curl http://192.168.200.104:8080/health

# Get capability profile
curl -s http://192.168.200.104:8080/cap

# Run singine Python CLI (if Python + singine installed in Termux)
pip install -e ~/ws/singine   # after git clone
singine cap detect
```

## Available endpoints

| Endpoint | Method | Description | Auth required |
|---|---|---|---|
| `/health` | GET | Ping — always returns 200 | No |
| `/cap` | GET | Machine capability profile (JSON) | No |
| `/messages` | GET | Mail search (`?search=invoice`) | No (dry-run) |
| `/loc/:iata` | GET | Resolve IATA → URN + timezone | No |
| `/timez` | GET | Timezone query (`?cities=BRU,NYC`) | No |
| `/backlog` | GET | Web backlog (when 16.H ships) | No |

## Network discovery

If you don't know the Mac's IP, use one of these:

```sh
# From Mac: find your LAN IP
ifconfig | grep "inet 192"

# From iOS (iSH/a-Shell): scan the LAN for port 8080
# (install nmap in iSH first: apk add nmap)
nmap -p 8080 192.168.200.0/24 --open
```

## Run at Mac startup (launchd)

To start singine server automatically when you log in:

```bash
# Create launch agent plist
cat > ~/Library/LaunchAgents/io.singine.server.plist << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>         <string>io.singine.server</string>
  <key>ProgramArguments</key>
  <array>
    <string>/usr/bin/env</string>
    <string>singine</string>
    <string>serve</string>
  </array>
  <key>WorkingDirectory</key> <string>/Users/skh/ws/singine</string>
  <key>RunAtLoad</key>        <true/>
  <key>KeepAlive</key>        <true/>
  <key>StandardOutPath</key>  <string>/tmp/singine-server.log</string>
  <key>StandardErrorPath</key><string>/tmp/singine-server.err</string>
</dict>
</plist>
EOF

# Load it
launchctl load ~/Library/LaunchAgents/io.singine.server.plist

# Check logs
tail -f /tmp/singine-server.log
```

## Security note

The server binds to `0.0.0.0` (all interfaces) on port 8080. It is accessible to anyone
on the same LAN. For sensitive endpoints (`/messages`), the trust model is:

- **No auth required** for read-only probe endpoints (`/health`, `/cap`, `/loc`, `/timez`)
- **Trust token** (future: Section 17 SSH-signed JWT) for write/mail endpoints
- **Firewall**: the Mac's built-in firewall (`System Settings → Network → Firewall`) is
  recommended for public networks; on home LAN it is not needed
